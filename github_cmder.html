Посредством чего работать с git/github:

Для работы с репозиториями git (в том числе и с github), на windows, можно использовать различные средства (посредством: https, ssh или скачивая архивы).

К таковым относятся:

  •  GUI Clients for git/github: GitHub for Windows, Git Extensions, SourceTree, git-cola, SmartGit, GitEye, etc [http://git-scm.com/downloads/guis]
  • console utils: git bash [https://msysgit.github.io/], putty [http://www.chiark.greenend.org.uk/~sgtatham/putty/download.html]

Рекомендуется использовать console utils - быстрее и интерфейс пользователя - единый - консоль.

Git Bash входим в пакет msysgit, который может быть включён во множество программ - используется для эмуляции работы unix (posix совместимых систем) - на windows.

Например, msysgit входит в состав окружения для консольной программы: cmder.


В состав cmder входят:
- ssh-agent
- ssh-keygen
- git bash

-----------------------------
Для того, чтобы подключиться к github репозиторию нам нужно сгенерировать ключи: публичный и приватный.
Публичный мы добавим в настройках подключения github. А приватный добавим в ssh-agent (в окружении msysgit).

-----------------------------

Как:

-- 1 -- Создаём каталог для ключей

1. Устанавливем cmder [http://gooseberrycreative.com/cmder/]
2. Заходим в cmder в домашний каталог пользователя (или любой другой, где собираетесь хранить ssh ключи). Например: C:\Users\%USERNAME%
3. Создаём каталог: ".ssh"

-- 2 -- Создание ssh ключей


4. В консоли cmder исполняем следующие команды:
- [ssh-keygen -t rsa -C "you@gmail.com"]. После этого в текущем каталоге будут созданы ключи. (you@gmail.com - заменить на Ваш email).
- [ssh -T git@github.com] - проверка подключения к gihub посредством ssh-agent.


-- 3 -- Добавление ssh ключей в ssh-agent 

- [ssh-add id_rsa] - добавить приватный ключ в ssh-agent (в окружении msysgit). Здесь id_rsa - имя файла приватного ключа.

-- 4 -- Добавление ssh ключей в Ваш аккаунт в репозиторий (github)

5. [git config --global user.email "you.github@gmail.com"] - добавим свой email (из аккаунта github)
6. В Ваших настройках на github заходим в Settings - SSH keys: Add SSH key.
Здесь:
- поле "Title" произвольно;
- поле "Key" - копируем содержимое файлы с публичным ключём (например: id_rsa.pub - открываем через любой блокнот - главное, чтобы файл назывался: *.pub).


Ваш клиент по работе с github настроен!

Переходим в каталог, где предполагаем работать с github.
И делаем [git clone], [git add .], [git commit -am 'fix'], [git push origin], ...
